# 3D Simulations

## Basic 3D FDTD

```python
from neurowave.core.config import GridConfig, SimulationConfig
from neurowave.solvers.fdtd_3d import FDTD3D
from neurowave.core.backend import set_backend

set_backend('cupy')  # Highly recommended for 3D

config = SimulationConfig(
    grid=GridConfig(
        nx=100, ny=100, nz=100,
        dx=1e-3, dy=1e-3, dz=1e-3
    ),
    total_steps=200
)

solver = FDTD3D(config=config, sources=[...], boundaries=[...])
solver.run()
```

## 3D Materials

```python
from neurowave.core.grid_3d import Grid3D

# Set a cuboid region
solver.grid.set_material_region(
    x_start=30, x_end=70,

    z_start=30, z_end=70,
    eps_r=4.0,
    sigma_e=0.1

```

## Memory Considerations

3D simulations use significantly more memory:

| Grid | Fields | Memory |
|------|--------|--------|
| 100x100x100 | 6 components + 4 materials | ~640 MB |
| 200x200x200 | 6 + 4 | ~5.1 GB |
| 300x300x300 | 6 + 4 | ~17 GB (needs V100/A100) |

For T4 (16 GB), stay under 200x200x200 per simulation.

## Performance

3D simulations benefit greatly from GPU — the compute-to-memory ratio is higher:

- CPU: ~1-5 Mcell-steps/s
- GPU (T4): ~50-200 Mcell-steps/s
- Speedup: 20-100x for large 3D grids
